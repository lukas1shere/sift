How Vector Databases Work

By Jane Developer · March 15, 2024 · 8 min read

Vector databases have become a foundational piece of the AI stack. If you're building a RAG pipeline, semantic search engine, or recommendation system, you're almost certainly reaching for one. But how do they actually work?

What Is a Vector?

A vector is simply an array of floating-point numbers — a point in high-dimensional space. Modern embedding models like OpenAI's text-embedding-3-small produce vectors with 1536 dimensions.

The key insight is that semantically similar content produces geometrically similar vectors. Two sentences about the same topic will have vectors that are close together in that 1536-dimensional space.

Similarity Search

The fundamental operation in a vector database is approximate nearest neighbor (ANN) search: given a query vector, find the K most similar vectors in the collection.

The most common similarity measures are:

  • Cosine similarity — angle between vectors. Most common for embeddings.
  • Dot product — fast but order-dependent on magnitude.
  • Euclidean distance — L2 distance. Useful when magnitude matters.

Indexing Algorithms

Brute-force comparison across millions of vectors is too slow. Vector databases use approximate indexing algorithms:

HNSW (Hierarchical Navigable Small World)

HNSW builds a layered graph where nodes are connected to their nearest neighbors. At query time, it navigates this graph greedily from the top layer down.

It offers excellent recall/speed tradeoff and is the default index in Qdrant, Weaviate, and most others.

IVF (Inverted File Index)

IVF clusters vectors into Voronoi cells. At query time, only the nearest clusters are searched. Used heavily in FAISS.

Here's how you'd set up a basic vector store with Python:

import qdrant_client as qc
from qdrant_client.models import Distance, VectorParams

client = qc.QdrantClient(":memory:")

client.create_collection(
    collection_name="docs",
    vectors_config=VectorParams(size=1536, distance=Distance.COSINE),
)

Metadata Filtering

Pure semantic search is rarely enough. You usually need to combine vector similarity with structured filters ("find the most similar docs where category = 'legal'").

This is called hybrid search and is supported natively in Qdrant, Pinecone, and Weaviate.
