Changelog

All notable changes to the Acme SDK are documented here. This project follows Semantic Versioning.

v2.3.0 — 2024-03-10

Added

  • New client.batch() method for parallel API calls
  • Support for streaming responses via stream: true option
  • Built-in retry logic with exponential backoff

Changed

  • Default timeout increased from 10s to 30s
  • client.process() now returns a typed response object instead of any

Fixed

  • Race condition in connection pool under high concurrency
  • Memory leak when streaming large responses

v2.2.1 — 2024-02-28

Fixed

  • Authentication headers were missing on retried requests
  • TypeScript types for metadata field now correctly typed as Record<string, unknown>

v2.2.0 — 2024-02-15

Added

  • Webhook signature verification utility: verifyWebhook(payload, secret)
  • New AcmeError class with structured error codes

Breaking Changes

  • Removed deprecated client.send() method. Use client.process() instead.

v2.1.0 — 2024-01-20

Added

  • Python SDK now available: pip install acme-sdk
  • Rate limit headers exposed in response object

v2.0.0 — 2024-01-01

Major release with a redesigned API surface.

Breaking Changes

  • API key must now be passed to constructor, not set globally
  • All methods are now async/await based
  • Minimum Node.js version bumped to 18
// Before (v1.x)
acme.setKey('sk-...');
const result = acme.process('hello');

// After (v2.x)
const client = new AcmeClient({ apiKey: 'sk-...' });
const result = await client.process({ input: 'hello' });
